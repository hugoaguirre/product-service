# Product (Catalog) Service

A small gRPC service written in Go that follows Hexagonal Architecture

---

## Quick overview

This service acts as a Catalog service that only manages
the `Product` entity in a system.

The idea behind this service is that other services can perform transactions
using the information managed by this service.

This service should not initiate workflows in other services.

## Adapters

### gRPC adapter

This adapter is meant to be used by other internal services, such as
an `Orders` service.

gRPC excels in having internal service communication language-agnostic
at high speed.

### REST adapter

Can be thought as the gateway of our system. This could be used by
the Frontend (web/mobile) or Admin Tools to browse or manage
the inventory respectively.

### Communication flow

- Web browser -> Product Service (`REST`) -- browsing the catalog
- Web browser -> Order Service (`REST`) -- placing an order
- Order Service -> Product Service (`gRPC`) -- internal validation during checkout

## Run it

The project's `Makefile` contains rules that simplifies the build and
run commands for you.

```shell
# Generate gRPC and SQL code
make generate

# Build and run
make build && make run
```

After a successful start, the service will span two servers (gRPC and REST)
listening on different ports.

## Test it

You can do basic cURL commands against the Product Service using its endpoints

```shell
curl localhost:8080/products/1
```
